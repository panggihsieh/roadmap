# 設計注意事項

## learningmap 學生頁捲動問題

### 問題現象
- `learningmap/index.html?view=student` 在學生模式下，目錄全部展開後無法往下捲動。
- 表面上看起來像是「右側沒有卷軸」，但實際上是整個學生頁的可捲動結構被版面配置吃掉了。

### 錯誤原因
- 原本版面採用 `grid + flex` 混合配置，並把主要內容區高度固定在 `calc(100vh - 85px)`。
- 學生模式下，樹狀目錄與右側詳細區又依賴內層面板自己的 `overflow-y: auto` 來捲動。
- 在某些瀏覽器與 viewport 條件下，這種「外層固定高度 + 內層滾動容器」的組合，會出現：
  - 元素 `scrollHeight` 與 `clientHeight` 被壓成一樣
  - 內層卷軸不出現
  - 使用者實際上完全無法向下瀏覽展開後的內容

### 額外踩到的問題
- GitHub Pages 端雖然已經部署新版 `style.css`，但 `learningmap/index.html` 仍可能吃到舊的 CSS 快取。
- 因此即使修正已經 push，上線頁面仍可能暫時看起來「完全沒變」。

### 最後採用的修正方式
- 學生模式改成「整頁捲動」優先，而不是依賴面板內部卷軸。
- 學生模式下：
  - `body.mode-student` 開啟 `overflow-y: auto`
  - `.main-container` 改為 `height: auto`
  - 學生模式的 `.panel-middle`、`.tree-root-container`、`.detail-body` 不再強依賴內層捲動
- `learningmap/index.html` 的 stylesheet 加上版本參數，避免正式站載入舊 CSS 快取。

### 後續設計準則
- 學生閱讀型頁面優先使用「整頁捲動」，不要預設使用巢狀卷軸。
- 只在工具面板、管理後台、或明確需要固定操作區時，才使用內層 `overflow`。
- 當 `grid` 或 `flex` 子元素需要可捲動時，務必檢查：
  - 是否需要 `min-height: 0`
  - 是否有外層固定 `100vh` / `calc(100vh - header)` 限制
  - 是否真的要讓捲動發生在子層，而不是整頁
- 靜態站台只要改到 CSS，若使用者反映「已 push 但頁面沒變」，要優先懷疑快取問題。
- 若是 GitHub Pages 或其他 CDN 靜態站台，建議為 CSS / JS 採用版本參數或檔名版號策略。
