# 即時互動投票操作說明

## 頁面入口

本機預覽網址：

```text
http://localhost:8081/
```

三個主要頁面：

- 觀眾投票頁：`http://localhost:8081/vote/poll/`
- 現場輸出頁：`http://localhost:8081/vote/output/`
- 主持人管理頁：`http://localhost:8081/vote/adm/`

## 觀眾投票頁操作

用途：給現場觀眾用手機或電腦投票。

操作方式：

1. 打開 `/vote/poll/`。
2. 閱讀目前題目。
3. 點選其中一個選項。
4. 投票後按鈕會顯示「已送出結果」，並顯示已投票狀態。
5. 若重新點選其他選項，會更新答案，不會新增第二票。
6. 若主持人關閉投票，選項會停用，不能再投票。
7. 頁面下方的 QR Code 可分享目前投票頁。

## 現場輸出頁操作

用途：給投影或現場大螢幕顯示即時結果。

操作方式：

1. 打開 `/vote/output/`。
2. 將這個頁面投影到現場螢幕。
3. 現場觀眾可以掃描畫面右上方 QR Code 進入投票頁。
4. 觀眾投票後，圓餅圖、票數與百分比會自動更新。
5. 主持人切換題目或清除票數後，畫面會同步更新。

## 主持人管理頁操作

用途：給主持人設定題目、開關投票、清除票數。

操作方式：

1. 打開 `/vote/adm/`。
2. 修改「活動名稱」。
3. 修改「題目」。
4. 在「選項」欄位輸入選項，一行一個。
5. 點「儲存題目」。
6. 點「開放投票」讓觀眾可以投票。
7. 點「關閉投票」停止投票。
8. 點「清除目前題目票數」重置目前題目的投票結果。

選項格式範例：

```text
Firebase 即時資料
前端互動畫面
部署與現場測試
```

## Demo 模式注意事項

目前若 `app.js` 裡沒有填入 Firebase config，系統會使用本機 demo 模式。

Demo 模式特性：

- 資料存在目前瀏覽器的 `localStorage`。
- 適合測試畫面與操作流程。
- 同一台瀏覽器的不同分頁可以同步。
- 不同手機或不同電腦不會真正同步。

若要讓現場觀眾手機與大螢幕同步，需要在 `app.js` 填入 Firebase Web App 設定，改用 Cloud Firestore。

## 現場測試流程

1. 主持人打開管理頁。
2. 現場螢幕打開輸出頁。
3. 觀眾打開投票頁。
4. 主持人確認題目與選項後，點「開放投票」。
5. 觀眾投票。
6. 檢查輸出頁票數是否即時更新。
7. 主持人點「關閉投票」。
8. 確認觀眾端不能再投票。
